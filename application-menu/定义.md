# Application Menu — 定义

---

## 是什么

Application Menu 是 macOS 顶部菜单栏（Menu Bar），应用级的功能入口和组织结构。

## UI 位置

屏幕最顶部，系统级，所有窗口共享。

## 职责

1. 定义应用的功能域分类
2. 每个功能域下组织具体操作（菜单项）
3. 绑定全局快捷键
4. 反映应用当前状态（菜单项根据上下文启用/禁用）

---

## 分类方法

Application Menu 由三类 Menu 组成：

### 1. 通用操作 Menu

与当前工作空间无关，任何时候都可用。例如：
- **Edit** — 撤销、复制、粘贴、查找
- **View** — 缩放、全屏、侧边栏显示/隐藏
- **Window** — 新建窗口、最小化、窗口排列

### 2. 工作空间 Menu

每种工作空间拥有一个独立 Menu，只在对应工作空间激活时有意义。例如：
- **Note** — 新建笔记、导出、格式化操作
- **PDF** — 书签、高亮、页面导航
- **Web** — 前进后退、提取内容

用户切换工作空间时，菜单栏自动反映当前上下文。

### 3. Help Menu

帮助、关于、快捷键参考、反馈入口。

---

## 设计原则

1. **零硬编码**：框架不硬编码任何 Menu 和 Menu Item，所有菜单通过注册机制注入
2. **框架只提供注册接口和渲染引擎**：框架不知道具体有哪些菜单，只负责接收注册、渲染菜单、派发事件
3. **插件注册工作空间 Menu**：新增工作空间（如未来的"画布"）= 插件注册一个新的 Menu，无需修改框架
4. **上下文感知**：工作空间 Menu 随当前活跃工作空间动态显示/隐藏或启用/禁用
