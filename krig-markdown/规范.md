# KRIG Markdown — 扩展格式规范

> **目的**：定义一种 Markdown 扩展格式，使 AI 能够无损地表达 KRIG-Note 的所有 Block 类型。
> 标准 Markdown 语法完全保留，KRIG 扩展语法仅用于标准 Markdown 无法表达的 Block 类型。

---

## 一、设计原则

1. **标准 Markdown 优先**：能用标准 Markdown 表达的，就用标准语法，不发明新语法
2. **扩展语法最小化**：只为标准 Markdown 无法表达的 Atom 类型添加扩展
3. **人可读**：扩展语法在纯文本编辑器中也应有合理的可读性
4. **无损往返**：KRIG Markdown → Atom JSON → KRIG Markdown，信息不丢失
5. **渐进增强**：不认识扩展语法的 Markdown 渲染器，仍能显示合理的降级内容

---

## 二、标准 Markdown 语法（直接映射）

以下语法与标准 Markdown 完全一致，无需扩展：

| Markdown 语法 | Atom 类型 | 说明 |
|---|---|---|
| `# 标题` | `partTitle` | 文档的第一个 H1 作为 partTitle |
| `## ~ ######` | `heading` | 2-6 级标题 |
| 普通段落 | `paragraph` | |
| `**粗体**` | mark: bold | |
| `*斜体*` | mark: italic | |
| `` `代码` `` | mark: code | |
| `~~删除线~~` | mark: strike | |
| `[文字](url)` | mark: link | |
| `> 引用` | `blockquote` | |
| `- 列表项` | `bulletList` + `listItem` | |
| `1. 列表项` | `orderedList` + `listItem` | |
| ` ``` ` 代码块 ` ``` ` | `code-block` | 含语言标识，也用于 ASCII 架构图 |
| `---` | `divider` | 水平分割线 |
| `\| 表格 \|` | `table` | 标准 Markdown 表格 |
| `![alt](src)` | `image` | |

## 三、扩展语法（KRIG 特有 Block）

### 3.1 容器语法：`:::type[params]`

用于有子内容的 Block 类型。三个冒号开始，三个冒号结束。

#### callout（提示框）

```
:::callout[NOTE]
提示框的内容，支持标准 Markdown 内联格式。
可以有多行内容。
:::
```

类型参数：`NOTE` | `WARNING` | `TIP` | `IMPORTANT` | `CAUTION`

→ Atom: `{ type: 'callout', content: { calloutType: 'note', children: [...] } }`

#### toggleHeading（可折叠标题）

```
:::toggle-heading[## 标题文字]
折叠区域内的内容。
可以包含任何标准 Block。
- 列表
- 也可以
:::
```

`[]` 内使用 Markdown heading 语法表示标题级别。

→ Atom: `{ type: 'toggleHeading', content: { level: 2, children: [...] } }`，子节点作为 children atoms

#### toggle-list（可折叠列表）

```
:::toggle-list
- :::toggle-item[标题文字]
  折叠内容
  :::
- :::toggle-item[另一个标题]
  另一段折叠内容
  :::
:::
```

→ Atom: `{ type: 'toggle-list' }`，每个 toggle-item 是子节点

#### column-list（多栏布局）

```
:::columns
:::col[50%]
左栏内容
:::
:::col[50%]
右栏内容
:::
:::
```

→ Atom: `{ type: 'column-list', content: { columns: 2 } }`

---

### 3.2 行内嵌入语法：`::type[content]{attrs}`

用于没有子内容的嵌入型 Block。两个冒号，不需要闭合。

#### video（视频嵌入）

```
::video[https://youtube.com/watch?v=xxx]{title="视频标题" duration="10:30"}
```

→ Atom: `{ type: 'video', content: { videoUrl: '...', title: '...', duration: '...' } }`

#### audio（音频嵌入）

```
::audio[https://example.com/audio.mp3]{title="音频标题"}
```

→ Atom: `{ type: 'audio', content: { audioUrl: '...', title: '...' } }`

#### tweet（推文嵌入）

```
::tweet[https://x.com/user/status/123456]
```

→ Atom: `{ type: 'tweet', content: { tweetUrl: '...', tweetId: '123456' } }`

#### note-link（笔记内链）

```
::note-link[note-abc-123]{title="另一篇笔记的标题"}
```

→ Atom: `{ type: 'note-link', content: { noteId: 'note-abc-123', title: '...' } }`

---

### 3.3 数学公式

使用已有约定（与 LaTeX / KaTeX 生态一致）：

#### math-block（块级公式）

```
$$
E = mc^2
$$
```

→ Atom: `{ type: 'math-block', content: { latex: 'E = mc^2' } }`

#### math-inline（行内公式）

```
行内公式 $E = mc^2$ 在段落中。
```

→ Atom: paragraph 中的 inline mark: `{ type: 'math-inline' }`

---

### 3.4 image 扩展属性

标准 Markdown image 语法可以附加属性：

```
![替代文字](图片地址){width="80%" caption="图片说明"}
```

→ Atom: `{ type: 'image', content: { src: '...', alt: '...', width: '80%', caption: '...' } }`

无扩展属性时等同标准 Markdown：
```
![替代文字](图片地址)
```

## 五、解析优先级

CLI 解析器按以下优先级匹配：

1. 容器块：`:::type[...]`（先匹配最外层容器）
2. 代码块：` ``` `（内部不解析任何扩展语法）
3. 数学块：`$$...$$`（内部不解析）
4. 行内嵌入：`::type[...]{...}`
5. 标准 Markdown 块级语法（heading、list、blockquote、table、hr）
6. 标准 Markdown 行内语法（bold、italic、code、link）
7. 数学行内：`$...$`
8. 纯文本段落

---

## 六、降级策略

当 KRIG Markdown 被不支持扩展语法的渲染器打开时：

| 扩展语法 | 降级显示 |
|---|---|
| `:::callout[NOTE]...\n内容\n:::` | 显示为 `:::callout[NOTE]` 文本 + 内容段落 |
| `::video[url]{title="..."}` | 显示为 `::video[url]{title="..."}` 纯文本 |
| `$$...$$` | 大多数 Markdown 渲染器已支持 |
| `![alt](src){width="80%"}` | 显示为普通图片（忽略 `{}`） |

> 这是引用文字

这是一个提示框，告诉读者一些有用的信息。
:::

:::toggle-heading[### 点击展开详情]
这里是折叠的内容。

- 可以包含列表
- 和其他 Block
:::

$$
\int_0^\infty e^{-x^2} dx = \frac{\sqrt{\pi}}{2}
$$

::video[https://youtube.com/watch?v=dQw4w9WgXcQ]{title="示例视频"}

::tweet[https://x.com/elonmusk/status/123456789]

参考另一篇笔记：::note-link[note-abc-123]{title="架构设计原则"}
