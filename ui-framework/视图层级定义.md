## 二、四层模型

应用从外到内共 **4 层**：

| 层级 | 名称 | 定义 | 个数 |
|------|------|------|------|
| **L0** | Application | macOS 桌面应用本身 | 1 |
| **L1** | Window (Shell) | 应用窗口 + 固定骨架 | 1 |
| **L2** | Workspace | 独立的工作环境 | N |
| **L3** | NavSide + Slot → View | 导航操作 + 内容渲染 | 每 Workspace 各 1 组 |

---

### 2.1 L0 — Application

| 属性 | 值 |
|------|------|
| **定义** | macOS 上运行的桌面应用 |
| **包含** | Application Menu + Window |

#### Application Menu

macOS 原生菜单栏，所有 Menu 和 Menu Item 通过注册机制注入，零硬编码。

| 分类 | 说明 | 实现方式 |
|------|------|---------|
| **通用操作 Menu** | Edit、View、Window 等，与工作空间无关 | 注册机制 |
| **工作空间 Menu** | 每种 Workspace 一个，随活跃 Workspace 动态显示 | 插件注册 |
| **Help Menu** | 帮助、关于、快捷键参考 | 注册机制 |

**详细定义**：`application-menu/定义.md`

---

### 2.2 L1 — Window (Shell)

| 属性 | 值 |
|------|------|
| **定义** | 应用的主窗口，同时也是固定骨架（Window = Shell，1:1 关系） |
| **包含** | WorkspaceBar + N 个 Workspace |

#### WorkspaceBar

| 属性 | 值 |
|------|------|
| **定义** | 管理 Workspace 生命周期的 Tab 栏 UI 控件 |
| **位置** | 窗口顶部 |
| **职责** | 创建、切换、关闭、排序 Workspace |

**WorkspaceBar 内部组件**：

| 组件 | 职责 |
|------|------|
| **SidebarToggle** | 展开/收起 NavSide（仅 NavSide 折叠时显示） |
| **WorkspaceLabel × N** | 工作空间标签（标题 + × 关闭按钮） |
| **CreateButton (+)** | 创建新工作空间 |

> **注意**：WorkspaceBar 只是管理 Workspace 的 UI 控件，不等于 Workspace 本身。

---

### 2.3 L2 — Workspace

| 属性 | 值 |
|------|------|
| **定义** | 独立的 KRIG-Note 工作环境。每个 Workspace 完全隔离 |
| **本质** | 逻辑实体，不是 UI 区域 |
| **显示** | WorkspaceBar 显示所有 Workspace 标签 |
| **隔离** | Workspace 之间的 View 实例完全独立，互不影响 |

```
Workspace 内部结构:
  ├── Layout Mode          当前的 Slot 划分方式（如 'note+thought'）
  ├── View 实例池          已创建的 View 实例（懒创建，每个 View 自带 Toolbar）
  │     ├── NoteView       (自带 NoteToolbar + Overlays)
  │     ├── PDFView        (自带 PdfToolbar + Overlays)
  │     ├── WebView        (自带 WebToolbar + Overlays)
  │     ├── ThoughtView    (+ Overlays)
  │     └── GraphView      (自带 GraphToolbar + Overlays)
  └── 驱动关系
        ├── → NavSide      驱动 NavSide 显示什么内容
        └── → Slot Area    驱动 Slot 显示什么布局
```

#### Workspace 操作

| 操作 | 触发 | 效果 |
|------|------|------|
| **创建** | 点击 WorkspaceBar [+] 或快捷键 | 创建新 Workspace 并切换 |
| **切换** | 点击 WorkspaceLabel 或快捷键 | hide 旧 Workspace 所有 View + show 新 Workspace 活跃 View |
| **关闭** | 点击 [×] 或快捷键 | 保存数据 → 销毁所有 View → 切到相邻 Workspace |
| **排序** | 拖拽 WorkspaceLabel | 改变 Workspace 顺序 |

#### Layout Mode（布局模式）

Layout Mode 决定 Workspace 如何划分 Slot，以及每个 Slot 放什么 View。

**命名规则**：`{leftView}+{rightView}` 或 `{view}-only`

**一等公民组合**（UI 直接支持，高频使用）：

| Layout Mode | Left Slot | Right Slot | 场景 |
|-------------|-----------|------------|------|
| `note-only` | NoteView | — | 专注写作 |
| `note+thought` | NoteView | ThoughtView | 编辑 + 批注 |
| `note+web` | NoteView | WebView | 编辑 + 网页/AI |
| `note+pdf` | NoteView | PDFView | 编辑 + PDF 参考 |
| `pdf-only` | PDFView | — | 专注阅读 |
| `pdf+note` | PDFView | NoteView | PDF 提取到笔记 |
| `pdf+thought` | PDFView | ThoughtView | PDF + 批注 |
| `pdf+web` | PDFView | WebView | PDF + AI 对话 |
| `web-only` | WebView | — | 全屏浏览 |
| `web+web` | WebView | WebView | 左右翻译对照 |
| `web+note` | WebView | NoteView | 浏览 + 记录 |

**二等公民组合**（可通过拖拽/快捷键触发，低频）：

| Layout Mode | Left Slot | Right Slot | 场景 |
|-------------|-----------|------------|------|
| `note+note` | NoteView | NoteView | 对照编辑 |
| `note+graph` | NoteView | GraphView | 编辑 + 图谱 |
| `graph-only` | GraphView | — | 图谱探索 |
| `thought-only` | ThoughtView | — | 全屏批注 |
| `pdf+pdf` | PDFView | PDFView | 对照阅读 |

---

### 2.4 L3 — NavSide + Slot → View

#### NavSide（导航操作区）

| 属性 | 值 |
|------|------|
| **定义** | 共享的渲染容器，内容由当前活跃 Workspace 驱动 |
| **位置** | 窗口左侧，可展开/收起 |

**NavSide 内部组件**：

| 区域 | 职责 | 实现方式 |
|------|------|---------|
| **Layout Toggle** | 展开/收起 NavSide | 框架硬编码 |
| **Brand Bar** | Logo + App Name + User Entry | 注册机制 |
| **ModeBar** | 模式切换（Note/PDF/Web/AI/Graph） | 注册机制（插件注册 Tab） |
| **Action Bar** | 当前模式的操作按钮（排序、新建等） | 注册机制（插件填充内容） |
| **Search** | 当前 Workspace 内搜索 | 插件硬编码 |
| **Content List** | 当前 Workspace 的内容列表 | 注册机制（插件渲染列表） |

NavSide 管理自己的 Overlays（右键菜单、Status Bar 浮窗、下拉选择器）。

**详细定义**：`navside/定义.md`

#### Slot（内容渲染区）

| 属性 | 值 |
|------|------|
| **定义** | Workspace 内的布局位置。一个 Slot 容纳一个 View |
| **种类** | Left Slot / Right Slot |
| **职责** | 决定 View 的位置和大小 |
| **约束** | 一个 Slot 同一时间只能放一个 View |

> **Slot 不是 View。** Slot 是"放 View 的位置"，View 是"放进去的内容"。

#### View（内容视图）

View 是最底层的内容单元。每种 View 由插件实现，框架定义 View 接口。

```
View（5 种类型，插件注册）
  ├── NoteView      笔记编辑器（自带 NoteToolbar + Overlays）
  ├── PDFView       PDF 阅读器（自带 PdfToolbar + Overlays）
  ├── WebView       Web 浏览器（自带 WebToolbar + Overlays）
  ├── ThoughtView   思考面板（+ Overlays）
  └── GraphView     知识图谱（自带 GraphToolbar + Overlays）
```

每个 View：
- 拥有自己的 **Toolbar**，在 View 内部渲染，不由外部管理
- 拥有自己的 **Overlays**（SlashMenu、FloatingToolbar、HandleMenu、右键菜单等）
- 通过框架定义的 **View 接口**与外部通信

## 三、View 之间的通信关系

View 之间不直接通信。所有跨 View 通信都经过 **main 进程路由**。

```
NoteView ──→ main ──→ ThoughtView    (创建/加载 thought)
PDFView  ──→ main ──→ ThoughtView    (创建/加载 thought)
PDFView  ──→ main ──→ NoteView       (提取内容到笔记)
WebView  ──→ main ──→ NoteView       (提取内容到笔记)
NoteView ──→ main ──→ WebView        (发送到 AI)
ThoughtView ──→ main ──→ NoteView    (滚动到锚点)
ThoughtView ──→ main ──→ PDFView     (滚动到高亮)
```

### 主要通信模式

| 模式 | 源 View | 目标 View | 数据流 | 触发方式 |
|------|---------|----------|--------|---------|
| **内容提取** | WebView / PDFView | NoteView | 结构化内容 | 用户选择区域 → AI 提取 |
| **思考联动** | NoteView / PDFView | ThoughtView | Thought 对象 | 用户创建批注 |
| **锚点跳转** | ThoughtView | NoteView / PDFView | 锚点 ID | 用户点击 thought |
| **AI 对话** | NoteView | WebView (AI) | 文本内容 | 用户发送选中内容 |

## 四、Overlay 归属

Overlay（浮窗）不是独立的层，而是每个子视图自己的能力：

| 子视图 | 它的 Overlays |
|--------|--------------|
| **Window (Shell)** | 全局级浮窗（如果有） |
| **NavSide** | 右键菜单、Status Bar 浮窗、下拉选择器 |
| **View** | SlashMenu、FloatingToolbar、HandleMenu、右键菜单 |
| **Toolbar** | 下拉菜单、颜色选择器、格式面板 |

**原则**：谁的浮窗谁管理。

## 五、Workspace 与 View 的生命周期

### 5.1 View 实例池

每个 Workspace 拥有自己的 View 实例池。View 懒创建，Workspace 关闭时全部销毁。

```
Workspace A (Layout: note+thought):
  View 实例池:
    ├── NoteView-A1    ← 活跃, Left Slot
    ├── ThoughtView-A1 ← 活跃, Right Slot
    ├── PDFView-A1     ← 已创建但隐藏（之前打开过 PDF）
    └── WebView        ← 未创建（从未使用过 Web 模式）

Workspace B (Layout: pdf+note):
  View 实例池:
    ├── PDFView-B1     ← 活跃, Left Slot
    ├── NoteView-B1    ← 活跃, Right Slot
    └── ThoughtView-B1 ← 已创建但隐藏
```

### 5.2 生命周期事件

```
View 创建 ── 用户首次切到需要该 View 的 Layout Mode
              → 懒创建 View 实例
              → 注册到 Workspace 的 View 实例池

View 隐藏 ── 切换 Layout Mode 或切换 Workspace
              → 隐藏但保留内部状态
              → 无需重新加载内容

View 显示 ── 切换回包含该 View 的 Layout Mode / Workspace
              → 显示并设置位置和大小
              → 内部状态完整保留

View 销毁 ── Workspace 关闭
              → 保存数据
              → 释放资源
```

## 六、架构约束（不变量）

1. **Workspace 隔离**：不同 Workspace 的 View 实例之间没有任何直接关联。切换 Workspace 只改变可见性，不改变任何 View 的内部状态。

2. **Toolbar 属于 View**：每个 View 内部自带 Toolbar，在 View 内部渲染。Workspace 和 Shell 不管理 Toolbar。

3. **Overlay 属于子视图**：每个子视图（NavSide、View、Toolbar）管理自己的浮窗。不存在全局 Overlay 管理层。

4. **Layout 与 View 解耦**：Slot 只管位置和大小，不关心里面放什么 View。任何 View 类型可以放到任何 Slot。

5. **View 懒创建**：View 实例在首次需要时才创建。Workspace 刚创建时可以没有任何 View 实例。

6. **View 生命周期绑定 Workspace**：View 随 Workspace 创建（懒），随 Workspace 关闭而销毁。不存在脱离 Workspace 独立存在的 View。

7. **跨 View 通信经过路由**：View 之间不直接通信，所有跨 View 消息经过 main 进程路由。

8. **注册优先**：Application Menu、NavSide 的大部分区域、View 类型均通过注册机制实现。只有框架自身行为（Layout Toggle）和基础能力（Search）硬编码。

9. **NavSide 是共享容器**：NavSide 的渲染实例只有一个，但内容由当前活跃 Workspace 驱动。切换 Workspace 时，NavSide 内容跟随切换。

## 七、命名规范

### 7.1 类型定义

```typescript
/** L3 — View 类型（插件注册） */
type ViewType = 'note' | 'pdf' | 'web' | 'thought' | 'graph';

/** L2 — 布局模式（{left}+{right} 或 {view}-only） */
type LayoutMode =
  | 'note-only' | 'pdf-only' | 'web-only' | 'graph-only' | 'thought-only'
  | `${ViewType}+${ViewType}`;

/** L3 — Slot 位置 */
type SlotSide = 'left' | 'right';

/** NavSide — 工作模式 */
type NavMode = 'note' | 'pdf' | 'web' | 'ai' | 'graph';
```

### 7.2 变量命名规范

| 概念 | 变量名 | 说明 |
|------|--------|------|
| NoteView 实例 | `noteView` | — |
| PDFView 实例 | `pdfView` | — |
| WebView 实例 | `webView` | — |
| ThoughtView 实例 | `thoughtView` | — |
| GraphView 实例 | `graphView` | — |
| 布局模式 | `layoutMode` | — |
| 工作模式 | `navMode` | NavSide 的 ModeBar 状态 |

## 八、层级关系图

```
L0  Application
     │
     ├── Application Menu (注册机制: 通用操作 + 工作空间 + Help)
     │
     └── L1  Window (Shell)
              │
              ├── WorkspaceBar (Tab 管理: 创建/切换/关闭 Workspace)
              │
              └── L2  Workspace × N
                       │
                       ├── NavSide (共享容器, 内容由活跃 Workspace 驱动)
                       │     ├── Layout Toggle (框架硬编码)
                       │     ├── Brand Bar (注册机制)
                       │     ├── ModeBar (注册机制: Note/PDF/Web/AI/Graph)
                       │     ├── Action Bar (注册机制: 标题 + 操作按钮)
                       │     ├── Search (插件硬编码)
                       │     ├── Content List (注册机制)
                       │     └── Overlays (NavSide 自管理)
                       │
                       └── L3  Slot Area
                              │
                              ├── Left Slot ──→ View + Toolbar + Overlays
                              ├── Divider
                              └── Right Slot ─→ View + Toolbar + Overlays
```
