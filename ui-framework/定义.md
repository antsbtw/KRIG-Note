# UI Framework — 定义

---

## 总体结构

KRIG-Note 的 UI 框架由 4 层组成：

```
macOS Application（Application Menu）
  └─ Window (Shell)
       └─ Workspace
            ├─ NavSide（+ 自己的 Overlays）
            └─ Slot（+ 自己的 Overlays）
                 └─ View + Toolbar（+ 自己的 Overlays）
```

---

## 各层定义

### L0: macOS Application

- **职责**：应用生命周期管理 + Application Menu
- **包含**：Application Menu（macOS 原生菜单栏）、一个或多个 Window
- **详细定义**：见 `application-menu/定义.md`

### L1: Window (Shell)

- **职责**：窗口骨架，承载全局共享的 UI 控件
- **包含**：
  - **WorkspaceBar**（Tab 栏）— 管理 Workspace 的创建、切换、关闭
  - **全局 Overlays** — 属于 Window 级别的浮窗
- **关键点**：一个 Window 就是一个 Shell，1:1 关系，不再区分

### L2: Workspace

- **职责**：独立工作环境，每个 Tab 对应一个 Workspace
- **本质**：逻辑实体，不是某个 UI 区域
- **驱动关系**：
  - 通过 **WorkMode**（注册制）驱动 NavSide 显示什么内容
  - 通过 **slotBinding** 驱动 Slot 显示什么 View
- **拥有的状态**：workModeId、slotBinding、分栏比例等
- **关键点**：WorkspaceBar 只是管理 Workspace 生命周期的 UI 控件，不等于 Workspace 本身
- **详细定义**：见 `ui-framework/workspace.md`、`ui-framework/workmode.md`

### L3: NavSide + Slot → View

**NavSide**（导航操作区）：
- 共享的渲染容器，内容由当前活跃 Workspace 的 WorkMode 驱动
- 包含：Layout Toggle、Brand Bar、ModeBar（插件注册 WorkMode Tab）、Action Bar、Search、Content List
- 每个子区域有自己的 Overlays（右键菜单、Status Bar 浮窗等）
- 详细定义：见 `navside/定义.md`

**Slot**（内容渲染区）：
- Left Slot + Divider + Right Slot
- 每个 Slot 放一个 View（纯布局位置，不感知 View 类型）

**View**（具体视图，由插件实现）：
- 内部结构统一：Toolbar + Content + Overlays
- 身份由 ViewType + variant 决定（4 种基础类型：note、pdf、web、graph）
- 实现框架定义的 View 接口（生命周期、状态持久化、焦点管理）
- View 之间默认孤立，有联系的 View 通过协同协议通信
- 详细定义：见 `ui-framework/view.md`、`ui-framework/view-protocol.md`

## Overlay 归属原则

Overlay（浮窗）不是独立的一层，而是每个子视图自己的能力：

| 子视图 | 它的 Overlays |
|--------|--------------|
| Window (Shell) | 全局级浮窗 |
| NavSide | 右键菜单、Status Bar 浮窗、下拉选择器 |
| View | SlashMenu、FloatingToolbar、HandleMenu、右键菜单 |
| Toolbar | 下拉菜单、颜色选择器 |

**原则**：谁的浮窗谁管理。框架为每个子视图提供 Overlay 的基础能力（层级管理、定位策略、生命周期），子视图自己决定何时显示什么浮窗。

---

## 通信机制

View 之间通过 main 进程路由通信，不直接通信：
- 框架只做消息路由，不理解消息内容
- View 间协同行为由**协同协议**查表匹配（translate / anchor / page-sync），默认无协同
- 详细定义：见 `ui-framework/view-protocol.md`
