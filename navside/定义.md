# NavSide — 定义

---

## 是什么

NavSide 是窗口左侧的导航操作区域。它是一个共享的渲染容器，内容由当前活跃 Workspace 的 **WorkMode** 驱动。

## 在框架中的位置

```
macOS Application
  └─ Window (Shell)
       └─ Workspace
            ├─ NavSide ← 这里
            └─ Slot → View
```

## 职责

1. 工作模式切换（ModeBar：插件注册 WorkMode Tab）
2. 当前 WorkMode 的操作入口（Action Bar、Search、Content List）
3. 用户入口（登录/账号/同步状态）
4. 面向用户操作的一致性——不管当前是哪个 WorkMode，交互模式保持一致，具体内容由插件注册

---

## 结构

```
┌─────────────────────────────┐
│ Layout Toggle               │  框架硬编码：展开/收起 NavSide
├─────────────────────────────┤
│ Brand Bar                   │  注册机制：Logo + App Name + User Entry
├─────────────────────────────┤
│ ModeBar                     │  注册机制：框架提供 Tab 容器，插件注册 WorkMode Tab
├─────────────────────────────┤
│ Action Bar                  │  注册机制：框架定义布局（左标题 + 右按钮），插件填充内容
├─────────────────────────────┤
│ Search                      │  插件硬编码：每个 Workspace 内置搜索，是基础能力
├─────────────────────────────┤
│ Content List                │  注册机制：当前 WorkMode 插件渲染内容列表
└─────────────────────────────┘
```

### NavSide 的 Overlays

NavSide 管理自己的浮窗，不依赖全局 Overlay 层：
- **Status Bar** — Toggle 浮窗，通过注册机制注入状态项
- **Context Menu** — 右键菜单，框架提供渲染和定位，插件注册菜单项
- **下拉选择器** — 排序方式、筛选条件等

---

## 各区域详细定义

### 1. Layout Toggle
- **实现方式**：框架硬编码
- **功能**：展开/收起 NavSide
- **说明**：这是框架自身的布局行为，不需要插件介入

### 2. Brand Bar
- **实现方式**：注册机制
- **布局**：左侧 Logo + App Name，右侧 User Entry
- **User Entry**：点击展开用户面板（登录/登出、账号设置、同步状态）；未登录时显示登录按钮
- **说明**：用户系统是全局的，不属于任何 Workspace

### 3. ModeBar
- **实现方式**：注册机制
- **功能**：切换当前 Workspace 的 WorkMode
- **框架职责**：提供 Tab 容器、切换动画、活跃状态管理
- **插件职责**：注册 WorkMode Tab（通过 `WorkModeRegistration`）
- **说明**：新增 WorkMode = 插件注册一个新 Tab，无需修改框架

### 4. Action Bar
- **实现方式**：注册机制
- **布局**：左侧标题（如"笔记目录"），右侧操作按钮（如"排序"、"+ 文件夹"、"+ 新建"）
- **框架职责**：定义左标题 + 右按钮的布局结构
- **插件职责**：通过 `ActionBarRegistration`（以 workModeId 为键）填充标题文字和操作按钮
- **说明**：切换 WorkMode 时，Action Bar 内容自动切换

### 5. Search
- **实现方式**：插件硬编码（每个 Workspace 的基础能力）
- **功能**：在当前 Workspace 内搜索（Note 搜笔记、PDF 搜文档、Web 搜书签）
- **说明**：搜索是 Workspace 的基础能力，不需要通过注册机制。全局跨 Workspace 搜索如果未来需要，走 Quick Actions（Cmd+K 命令面板），不放在 NavSide 里

### 6. Content List
- **实现方式**：注册机制
- **功能**：展示当前 WorkMode 的内容（Note 显示笔记树、PDF 显示文档列表、Web 显示书签）
- **框架职责**：提供可滚动的容器区域
- **插件职责**：通过 `ContentListRegistration`（以 workModeId 为键）渲染具体的列表内容、树形结构、拖拽排序等

---

## 注册接口

NavSide 的内容注册以 **workModeId** 为键（不是 ViewType），因为同一 ViewType 的不同 variant 可能需要不同的 NavSide 内容。

### WorkMode Tab 注册

```typescript
interface WorkModeRegistration {
  id: string;                // 唯一标识，如 'note', 'ai', 'thought'
  viewType: ViewType;        // 打开什么 View
  variant?: string;          // 什么变体（可选）
  icon: IconDefinition;      // ModeBar Tab 图标
  label: string;             // ModeBar Tab 文字标签
  order: number;             // ModeBar 排列顺序（数字越小越靠前）
}
```

### Action Bar 注册

```typescript
interface ActionBarRegistration {
  workModeId: string;           // 挂在哪个 WorkMode 下
  title: string;                // 左侧标题（如"笔记目录"）
  actions: ActionDefinition[];  // 右侧操作按钮
}

interface ActionDefinition {
  id: string;
  icon: IconDefinition;
  label: string;                // tooltip
  handler: () => void;
}
```

### Content List 注册

```typescript
interface ContentListRegistration {
  workModeId: string;                        // 挂在哪个 WorkMode 下
  render: (container: HTMLElement) => void;   // 渲染到框架提供的容器
  dispose: () => void;                        // 清理
}
```

## WorkMode 驱动 NavSide 的机制

```
用户在 ModeBar 切换 WorkMode（或切换 Workspace）
  │
  ├── ModeBar 切换活跃 Tab
  ├── Action Bar 切换到该 workModeId 注册的标题和操作按钮
  ├── Content List 切换到该 workModeId 注册的渲染器
  └── Search 切换搜索上下文
```

**详细定义（WorkMode 注册制）**：`ui-framework/workmode.md`

---

## 设计原则

1. **注册优先**：除 Layout Toggle（框架硬编码）和 Search（插件硬编码）外，所有区域通过注册机制实现，修改和扩展不需要改框架代码
2. **交互一致性**：框架统一定义各区域的交互模式（Tab 切换、Action Bar 布局、Content List 滚动），保证用户在不同 WorkMode 下操作体验一致
3. **内容由插件决定**：框架不知道有哪些 WorkMode、有哪些操作按钮、有哪些列表内容——这些全部由插件注册
4. **WorkMode 驱动内容切换**：切换 WorkMode 时，Action Bar、Search、Content List 自动切换为对应 WorkMode 的内容
5. **Overlay 自管理**：NavSide 管理自己的浮窗（右键菜单、Status Bar、下拉选择器），不依赖全局 Overlay 层
6. **内容挂 workModeId**：ActionBar 和 ContentList 的注册以 workModeId 为键，不以 ViewType 为键，支持同一 ViewType 的不同 variant 有独立的 NavSide 内容
