# NavSide — 定义

---

## 是什么

NavSide 是窗口左侧的导航操作区域。它是一个共享的渲染容器，内容由当前活跃的 Workspace 驱动。

## 在框架中的位置

```
macOS Application
  └─ Window (Shell)
       └─ Workspace
            ├─ NavSide ← 这里
            └─ Slot → View
```

## 职责

1. 工作空间模式切换（ModeBar：Note/PDF/Web/AI/Graph）
2. 当前工作空间的操作入口（Action Bar、Search、Content List）
3. 用户入口（登录/账号/同步状态）
4. 面向用户操作的一致性——不管当前是哪个工作空间，交互模式保持一致，具体内容由插件注册

---

## 结构

```
┌─────────────────────────────┐
│ Layout Toggle               │  框架硬编码：展开/收起 NavSide
├─────────────────────────────┤
│ Brand Bar                   │  注册机制：Logo + App Name + User Entry
├─────────────────────────────┤
│ Workspace Tabs (ModeBar)    │  注册机制：框架提供 Tab 容器，插件注册 Tab
├─────────────────────────────┤
│ Workspace Action Bar        │  注册机制：框架定义布局（左标题 + 右按钮），插件填充内容
├─────────────────────────────┤
│ Workspace Search            │  插件硬编码：每个 Workspace 内置搜索，是基础能力
├─────────────────────────────┤
│ Workspace Content List      │  注册机制：当前 Workspace 插件渲染内容列表
└─────────────────────────────┘
```

### NavSide 的 Overlays

NavSide 管理自己的浮窗，不依赖全局 Overlay 层：
- **Status Bar** — Toggle 浮窗，通过注册机制注入状态项
- **Context Menu** — 右键菜单，框架提供渲染和定位，插件注册菜单项
- **下拉选择器** — 排序方式、筛选条件等

---

## 各区域详细定义

### 1. Layout Toggle
- **实现方式**：框架硬编码
- **功能**：展开/收起 NavSide
- **说明**：这是框架自身的布局行为，不需要插件介入

### 2. Brand Bar
- **实现方式**：注册机制
- **布局**：左侧 Logo + App Name，右侧 User Entry
- **User Entry**：点击展开用户面板（登录/登出、账号设置、同步状态）；未登录时显示登录按钮
- **说明**：用户系统是全局的，不属于任何 Workspace

### 3. Workspace Tabs (ModeBar)
- **实现方式**：注册机制
- **功能**：切换当前 Workspace 的模式（Note/PDF/Web/AI/Graph）
- **框架职责**：提供 Tab 容器、切换动画、活跃状态管理
- **插件职责**：注册 Tab（图标 + 标签），例如 Note、PDF、Web、AI、Graph
- **说明**：新增工作空间模式 = 插件注册一个新 Tab，无需修改框架

### 4. Workspace Action Bar
- **实现方式**：注册机制
- **布局**：左侧标题（如"笔记目录"），右侧操作按钮（如"排序"、"+ 文件夹"、"+ 新建"）
- **框架职责**：定义左标题 + 右按钮的布局结构
- **插件职责**：填充标题文字和操作按钮
- **说明**：随 Workspace 模式切换，Action Bar 内容自动切换

### 5. Workspace Search
- **实现方式**：插件硬编码（每个 Workspace 的基础能力）
- **功能**：在当前 Workspace 内搜索（Note 搜笔记、PDF 搜文档、Web 搜书签）
- **说明**：搜索是 Workspace 的基础能力，不需要通过注册机制。全局跨 Workspace 搜索如果未来需要，走 Quick Actions（Cmd+K 命令面板），不放在 NavSide 里

### 6. Workspace Content List
- **实现方式**：注册机制
- **功能**：展示当前 Workspace 的内容（Note 显示笔记树、PDF 显示文档列表、Web 显示书签）
- **框架职责**：提供可滚动的容器区域
- **插件职责**：渲染具体的列表内容、树形结构、拖拽排序等

---

## 设计原则

1. **注册优先**：除 Layout Toggle（框架硬编码）和 Workspace Search（插件硬编码）外，所有区域通过注册机制实现，修改和扩展不需要改框架代码
2. **交互一致性**：框架统一定义各区域的交互模式（Tab 切换、Action Bar 布局、Content List 滚动），保证用户在不同 Workspace 下操作体验一致
3. **内容由插件决定**：框架不知道有哪些 Workspace 模式、有哪些操作按钮、有哪些列表内容——这些全部由插件注册
4. **Workspace 驱动内容切换**：切换模式时，Action Bar、Search、Content List 自动切换为对应 Workspace 的内容
5. **Overlay 自管理**：NavSide 管理自己的浮窗（右键菜单、Status Bar、下拉选择器），不依赖全局 Overlay 层
